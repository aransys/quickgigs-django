# Changelog

## 2.0.0 — Complete overhaul

Total rewrite of the structure, security posture, and UI. The model layer is
backward-compatible (with new migrations); the templates and CSS are not.

### Removed

- Committed virtual environments (`.venv/`, `venv_local/`).
- Committed `.env` with live Stripe TEST keys. **You should rotate those keys
  in the Stripe dashboard — they remain in git history.**
- `app.py` WSGI-monkey-patching hack with hardcoded Render hostname.
- `staticfiles/` collected output (regenerated by `collectstatic`).
- 122 KB README, 335 KB design.md, 76 KB code_quality.md, three Testing*.md
  files, and the `docs/screenshots/` tree.
- Bespoke 3,741-line CSS (5 files) replaced by Tailwind utilities.
- 912-line `gigs/templates/gigs/base.html` replaced by `templates/base.html`.
- Leftover `Task` model templates from the project's todo-app origin.

### Changed

- Django **4.2 → 5.1.4**.
- Settings — single env-driven module using `django-environ`. No
  platform-specific branches, no hardcoded hostnames. Security headers (HSTS,
  secure cookies, etc.) automatically engage when `DJANGO_DEBUG=False`.
- Database driver `psycopg2-binary → psycopg[binary] 3.x`.
- `Gig` model: typed `Category` choices, optional `slug` (with autogeneration),
  `featured_until` field, `MinValueValidator` on budget, additional indexes.
- `Application` model: typed `Status` choices, `UniqueConstraint` (replaces
  `unique_together`), additional indexes.
- `Payment` model: split `stripe_payment_id` into `stripe_session_id` plus
  `stripe_payment_intent`, added `currency`, typed `Type` and `Status` enums.
- Replaced ad-hoc `PaymentHistory` with `PaymentEvent` keyed on the Stripe
  event ID for idempotent webhook processing.
- `UserProfile.user_type` now includes an explicit `unset` choice so new
  signups don't default to "freelancer" silently.
- Profile-creation signal moved into `accounts/signals.py` and wired through
  `AccountsConfig.ready()`.
- Login redirect: `LOGIN_REDIRECT_URL = gigs:gig_list`,
  `LOGOUT_REDIRECT_URL = core:home`.
- Tests rewritten to use pytest-django with fixtures (see `conftest.py`).
- README rewritten from scratch.

### Added

- `templates/` (project-level base + partials: navbar, messages, footer,
  form-field).
- `templates/base.html` — Tailwind via play CDN with a custom config
  (brand-green palette, Inter font), HTMX, Alpine.js.
- `core/context_processors.py` — exposes site name, payments flag, etc. to
  every template.
- Idempotent Stripe webhook handler that's the **only** source of truth for
  flipping a gig to featured (the success page is now purely a UX
  confirmation).
- `Dockerfile` (multi-stage, non-root) + `docker-compose.yml` (web + Postgres).
- `scripts/entrypoint.sh` — handles `web`, `web-dev`, `migrate`, `manage`,
  `bash` modes.
- `scripts/cleanup-legacy.sh` — one-shot remover of pre-overhaul files.
- `Makefile` with sensible developer shortcuts (`make help`).
- `pyproject.toml` for ruff, pytest, coverage.
- `requirements-dev.txt`.
- `.env.example` documenting every configuration knob.

### Security

- Removed leaked Stripe keys from `.env` (still in git history — rotate them).
- Fixed `.gitignore` typo (`.env.env` → `.env`).
- HSTS, secure cookies, `SECURE_PROXY_SSL_HEADER`, sane CSRF settings — all
  automatic when DEBUG is off.
- Webhook now verifies signatures and is idempotent.
- `update_application_status`, `gig_applications`, etc. return 403 instead of
  redirect-with-message on permission failure (no ambiguity).
