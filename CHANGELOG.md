# Changelog

## 2.0.0 — Complete overhaul

Full rewrite of the structure, security posture, and UI. Live at
<https://quickgigs-tblp.onrender.com>.

### Removed

- Committed virtual environments (`.venv/`, `venv_local/`).
- Committed `.env` containing live Stripe TEST keys (rotated and scrubbed
  from history via `git-filter-repo`).
- `app.py` WSGI-monkey-patching hack with hardcoded Render hostname.
- `staticfiles/` collected output (regenerated by `collectstatic`).
- 122 KB README, 335 KB design.md, 76 KB code_quality.md, three Testing*.md
  files, and the `docs/screenshots/` tree.
- 3,741 lines of bespoke CSS across 5 files — replaced by Tailwind utilities.
- 912-line `gigs/templates/gigs/base.html` — replaced by
  `templates/base.html`.
- Leftover `Task` model templates from the project's todo-app origin.

### Changed

- Django **4.2 → 5.1.4**, single env-driven settings module
  (`django-environ`). Security headers (HSTS, secure cookies, etc.)
  automatically engage when `DJANGO_DEBUG=False`. No platform-specific
  branches or hardcoded hostnames.
- Database driver `psycopg2-binary → psycopg[binary] 3.x`.
- `Gig` model: typed `Category` choices, added `featured_until`,
  `MinValueValidator` on `budget`, additional indexes.
- `Application` model: typed `Status` choices, `UniqueConstraint`
  replacing `unique_together`, additional indexes.
- `Payment` model: split `stripe_payment_id` into `stripe_session_id` +
  `stripe_payment_intent`, added `currency`, typed `Type` and `Status`
  enums.
- Replaced ad-hoc `PaymentHistory` model with `PaymentEvent` keyed on the
  Stripe event ID for idempotent webhook processing.
- `UserProfile.user_type` now includes an explicit `unset` choice so new
  signups don't silently default to "freelancer".
- Profile-creation signal moved into `accounts/signals.py` and wired
  through `AccountsConfig.ready()`.
- Stripe webhook is now the **only** source of truth for marking gigs as
  featured. The success page is a UX confirmation only.
- Tests rewritten with pytest-django + fixtures (see `conftest.py`).
- README rewritten from scratch.

### Added

- `templates/` (project-level `base.html` + partials: navbar, messages,
  footer, form-field).
- Tailwind via Play CDN with a custom config (brand-green palette, Inter
  font), HTMX, Alpine.js.
- `core/context_processors.py` — exposes site name, payments flag, etc. to
  every template.
- Idempotent Stripe webhook handler with signature verification.
- `Dockerfile` (multi-stage, non-root) + `docker-compose.yml`
  (web + Postgres).
- `render.yaml` — Render Blueprint for one-click deploy.
- `scripts/entrypoint.sh` — handles `web`, `web-dev`, `migrate`,
  `manage`, `bash` modes.
- `Makefile` with sensible developer shortcuts (`make help`).
- `pyproject.toml` for ruff, pytest, coverage.
- `requirements-dev.txt`.
- `.env.example` documenting every configuration knob.

### Security

- Removed the leaked Stripe TEST keys, rotated them in Stripe, and scrubbed
  `.env` from git history.
- Fixed `.gitignore` typo (`.env.env` → `.env`).
- HSTS, secure cookies, `SECURE_PROXY_SSL_HEADER`, sane CSRF settings —
  automatic when `DEBUG` is off.
- Webhook verifies signatures and is idempotent via `PaymentEvent` unique
  on Stripe event ID.
- View-level permission failures return 403 instead of redirect-with-message
  (no ambiguity for clients).

### Schema migrations

- `gigs/migrations/0006_modernize_gig_application.py` —
  adds `featured_until` to Gig, replaces Application's `unique_together`
  with a named UniqueConstraint, adds Application indexes.
- `gigs/migrations/0007_alter_gig_options_and_more.py` —
  picks up validator/help-text drift from `makemigrations`.
- `accounts/migrations/0004_modernize_userprofile.py` and
  `0005_alter_userprofile_options_alter_userprofile_bio_and_more.py`.
- `payments/migrations/0002_modernize_payments.py`.
